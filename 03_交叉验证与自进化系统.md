# 03 内容×投放 交叉验证与自进化系统

> 本文档定义：如何让内容越做越好、投放越投越准
> 核心机制：每日微调 → 每周迭代 → 每阶段进化
> 目标：从"人指挥AI干"进化到"数据指挥AI自己干"

---

## 一、核心理念：OODA循环

整套系统运行在一个军事级OODA决策循环上：

```
Observe（观察）→ Orient（判断）→ Decide（决策）→ Act（行动）
    ↑                                                    |
    └────────────── 数据回流 ──────────────────────────────┘
```

每天跑一个小循环（微调），每周跑一个中循环（迭代），每阶段跑一个大循环（进化）。系统的"智能"不来自任何单一AI工具，而来自这个循环转得够快、反馈够准。

## 二、每日交叉验证（小循环：24小时）

### 2.1 每日必做：内容×投放交叉分析

每晚22:00-23:00，用以下矩阵交叉分析当天所有内容：

```
              │  自然流表现好     │  自然流表现差
──────────────┼─────────────────┼──────────────────
投流ROI好      │  A区：超级爆款    │  B区：花钱能跑
              │  → 全力追投+复制  │  → 谨慎投，测天花板
──────────────┼─────────────────┼──────────────────
投流ROI差/未投  │  C区：自然爆款    │  D区：无效内容
              │  → 立刻投流放大   │  → 淘汰，分析原因
```

**A区内容（自然好+投流好）：** 这是你的印钞机。立刻做5个以上变体（换hook/换配音/换封面），明天全部投放。单条追投上限提高到2000元。同时分析它为什么好——是hook吸引？是产品展示方式？是配音风格？把这些要素提取出来写入"爆款基因库"。

**B区内容（自然差+投流好）：** 说明内容本身平台不推荐，但目标人群精准。优化方向：改hook提高自然曝光，保持正文卖点不变。

**C区内容（自然好+未投流）：** 说明内容有吸引力但没拿到付费流量。立刻补投100元测试，大概率会进入A区。

**D区内容（自然差+投流差或未投）：** 直接淘汰。但在淘汰前记录一条：它的hook是什么？模板是什么？产品是什么？这些信息进入"失败案例库"，避免重复犯错。

### 2.2 每日交叉验证表

| 维度 | 具体对比项 | 看什么 |
|------|---------|--------|
| **模板×产品** | 同一模板推不同产品 | 哪个产品配哪个模板出单最多 |
| **hook×完播率** | 不同hook的完播率对比 | 哪类开头最留人 |
| **账号×转化率** | 同一内容发在不同账号 | 哪个人设的粉丝更愿意买 |
| **时段×播放量** | 同类内容不同时间发 | 什么时间发播放量最高 |
| **平台×客单价** | 同一产品在三个平台的销售 | 哪个平台卖高客单最好 |
| **AI纯度×限流** | 纯AI vs 混合实拍 | AI含量多高会被限流 |

### 2.3 每日输出物

每晚复盘结束后，必须产出一份"次日生产指令"：

```
【次日生产指令 - Day X+1】

1. 放大方向：
   - 模板_{X} + 产品_{Y} 的组合今天表现最好，明天产量提升到40%
   - hook风格：{共鸣型/震惊型/好奇型} 完播率最高

2. 测试方向：
   - 尝试把A区爆款的hook嫁接到新产品上
   - 测试{新场景/新人设/新时段}

3. 淘汰方向：
   - 模板_{Z} 连续3天D区，明天停止生产
   - {某个标签组合} 无效，换新标签

4. 投流指令：
   - 追投ID: {内容ID列表}，各追{金额}
   - 新投测试：今天的C区内容{ID列表}，各投100元

5. 特殊提醒：
   - {如：某账号疑似被限流，明天减少发布量观察}
```

## 三、每周迭代（中循环：7天）

### 3.1 每周六晚：战略复盘会（30分钟）

**复盘议程：**

```
1. 数据总览（5分钟）
   - 本周GMV vs 目标
   - 本周内容总量 / 出单内容占比
   - 本周投流总额 / 整体ROI

2. 爆款拆解（10分钟）
   - 本周TOP5内容详细分析
   - 提取"爆款基因"：hook公式、画面节奏、配音风格、产品展示方式
   - 这些基因写入"爆款基因库"

3. 失败分析（5分钟）
   - 本周BOTTOM5内容为什么失败？
   - 是否有系统性问题（如某个模板整体不行、某个产品卖不动）

4. 下周策略调整（10分钟）
   - 模板配比调整
   - 产品推力调整
   - 投流策略调整
   - 人员/账号调整（如果某人某平台表现特别好，加大他的产出）
```

### 3.2 周迭代的关键动作

**内容进化：**
- 把本周A区内容的共性提取为"模板升级版"
- 新模板 = 旧模板骨架 + 本周验证过的爆款基因
- 下周40%产量用升级版模板，10%用全新实验

**投流进化：**
- 统计本周各时段的投流ROI → 确定"黄金投放时段"
- 统计各投放金额档位的效率 → 找到"最优首投金额"
- 下周投流策略根据数据微调，不拍脑袋

**人设进化：**
- 哪个人的哪个账号出单最好？为什么？
- 是否需要调整某人的人设方向？
- 某人如果在某平台表现特别突出，考虑让他"偏科"集中精力

## 四、阶段进化（大循环：7天×3）

### 4.1 三个阶段的进化逻辑

```
第一阶段（Day 1-7）：探测
  核心能力：发现什么能跑
  进化产出：3-5个有效模板 + 初步数据基线
  ↓
  阶段进化：淘汰无效模板，锁定方向

第二阶段（Day 8-14）：精炼
  核心能力：把"能跑"优化成"跑得好"
  进化产出：爆款基因库 + 投流精准模型 + 最优产品组合
  ↓
  阶段进化：从模板复制升级为模板变异（自动生成更好的变体）

第三阶段（Day 15-21）：收割
  核心能力：把所有验证过的东西推到极限
  进化产出：端到端自动化流程 + 完整数据资产
  ↓
  项目进化：整套系统可复用于下一个品牌
```

### 4.2 阶段间的策略跃迁

**第一阶段→第二阶段的跃迁（Day 7 执行）：**

| 项目 | 第一阶段做法 | 第二阶段做法 |
|------|-----------|-----------|
| 内容生产 | 5个模板平均分配 | 集中80%产能到TOP3模板 |
| 投流 | 小额广撒测试 | 中额集中投验证过的内容 |
| 产品 | 以梨膏糖引流为主 | 转向罐装饮料冲GMV |
| 质检 | 每条人工检查 | 模板化生产+抽检 |
| 数据 | 记录为主 | 数据驱动决策 |

**第二阶段→第三阶段的跃迁（Day 14 执行）：**

| 项目 | 第二阶段做法 | 第三阶段做法 |
|------|-----------|-----------|
| 内容生产 | 模板批量复制 | 爆款基因重组+极限产出 |
| 投流 | 验证后追投 | 只投A区内容，预算全开 |
| 产品 | 主推罐装饮料 | 组合推荐拉高客单价 |
| 话术 | 日常种草 | 限时+紧迫感+促销 |
| 私域 | 开始沉淀 | 裂变冲量 |

## 五、自进化引擎：爆款基因库

### 5.1 什么是爆款基因库？

每次出现A区内容（自然好+投流好），把它的成功要素拆解成独立的"基因片段"存入库中。

```json
{
  "基因ID": "G-007",
  "来源内容ID": "D5-DY-HX-03",
  "基因类型": "hook公式",
  "基因描述": "以'王老吉竟然还有...'的震惊发现式开头",
  "适用平台": "抖音",
  "适用产品": "全品",
  "验证数据": {
    "使用次数": 12,
    "平均完播率": "32%",
    "平均转化率": "3.8%",
    "成功率": "58%"
  },
  "组合建议": "搭配G-003(产品特写展示)效果最佳"
}
```

基因类型包括：
- **hook基因** — 有效的开头公式
- **节奏基因** — 有效的视频节奏（如"3s悬念+10s展示+5s促单"）
- **视觉基因** — 有效的画面风格（如"金色产品+红色背景"）
- **配音基因** — 有效的声音风格
- **CTA基因** — 有效的转化话术
- **标签基因** — 有效的标签组合

### 5.2 基因重组：自动生成更好的内容

当基因库积累到20个以上基因片段后，可以用Claude自动进行"基因重组"：

```
系统指令：
"基于以下爆款基因库，生成10条新脚本。
要求：每条脚本至少组合3个不同类型的A级基因。
优先使用验证成功率>50%的基因。
尝试将高成功率的hook基因与新产品组合。"

输入：爆款基因库JSON
输出：10条新脚本（每条标注使用了哪些基因）
```

这就是"自进化"——不是人去想下一条视频拍什么，而是系统基于历史数据自动组合出更有可能成功的内容方案。

### 5.3 投放精准度自进化

**第一周：** 投放完全靠规则（完播率>25%就投100元）
**第二周：** 有了数据后，建立"出单预测模型"：

```
出单概率评分 = 
  完播率权重(0.3) × 完播率得分 +
  点击率权重(0.25) × 商品点击率得分 +
  互动率权重(0.15) × (点赞+评论+转发)/播放 +
  历史模板权重(0.2) × 该模板历史出单率 +
  时段权重(0.1) × 该时段历史转化率

评分 > 70分 → 立刻投200元
评分 50-70 → 投100元观察
评分 < 50 → 不投
```

**第三周：** 模型经过两周数据训练后，权重会更准确。每天根据实际出单数据回调权重：

```
如果实际出单率比预测高20% → 该内容的特征权重上调
如果实际出单率比预测低20% → 该内容的特征权重下调
```

## 六、长程时序规划：21天节奏控制

### 6.1 内容策略的时间维度

不是每天都做一样的事。21天有节奏感：

```
Day 1-3:  认知建立期 — 让平台和用户认识你
          内容重心：品牌故事、科普、不带货的好内容
          产品：不强推

Day 4-7:  信任积累期 — 开始软性带货
          内容重心：开箱测评、体验分享、场景种草
          产品：梨膏糖(28.8)为主，低门槛建立购买信任

Day 8-12: 转化爬坡期 — 全面带货
          内容重心：场景痛点、品牌背书、限时推荐
          产品：罐装饮料(88)为主力

Day 13-16: 客单攀升期 — 拉高客单价
          内容重心：送礼场景、组合推荐、礼盒对比
          产品：加大礼盒(158/168)推力

Day 17-19: 紧迫收割期 — 制造紧迫感
          内容重心：限时优惠、最后机会、库存告急
          产品：全品线冲量

Day 20-21: 终极收官 — 最后冲刺
          内容重心：回顾+最后一波促销
          产品：哪个卖得好就主推哪个
```

### 6.2 投放策略的时间维度

```
Day 1-4:  不投流 / 极小额测试（日<2000元）
          目的：积累自然流数据基线

Day 5-7:  小额分散投（日3000-5000元）
          目的：验证哪些内容值得投

Day 8-14: 中额集中投（日5000-8000元）
          目的：放大验证过的内容

Day 15-19: 大额精准投（日8000-12000元）
          目的：ROI最大化，冲GMV

Day 20-21: 弹性投放（剩余预算全部释放）
          目的：最后收割
```
