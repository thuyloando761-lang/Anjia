# 05 T+1内容库存与品牌审核SOP

> 核心约束：品牌方审核周期24小时
> 这意味着：Day N 生产的内容 → Day N 晚提交审核 → Day N+1 发布
> 本文档重新定义了整个团队的每日时间线

---

## 一、T+1 如何改变一切

### 1.1 之前的假设（错误）
```
9:00 AI生产内容 → 12:00 发布第一波 → 19:00 发布第二波
（同日生产同日发布）
```

### 1.2 现实约束下的正确流程
```
Day N
  9:00-14:00  AI生产 Day N+1 的内容
  14:00-15:00 内部质检
  15:00       提交品牌方审核
  （品牌方审核中，最长24小时）

Day N+1
  9:00-10:00  接收品牌方审核结果
  10:00-11:00 修改被打回的内容 → 紧急重审或替换为备用内容
  12:00       发布第一波（已审核通过的内容）
  19:00       发布第二波
```

### 1.3 关键结论

- **每天上午你生产的不是"今天发的内容"，而是"明天发的内容"**
- 你必须保持至少1天的"内容库存"——如果明天的内容被品牌方全部打回，你还有备用的可以发
- Day 1 实际上是"纯生产日"，因为你Day 1生产的内容要Day 2才能发布
- 这意味着实际发布周期是Day 2 - Day 23（但Day 1就要开始生产Day 2的内容）

## 二、每日时间线（T+1版本）

### 2.1 完整日程

| 时间 | 任务 | 说明 |
|------|------|------|
| **9:00-10:00** | 接收品牌方审核结果 | 查看昨天提交的内容哪些通过、哪些被打回 |
| **10:00-11:00** | 处理打回内容 | 修改后紧急重新提交，或用备用内容替换 |
| **11:00-11:30** | 确认今日发布清单 | 只发布已审核通过的内容 |
| **12:00-13:00** | 第一波发布 | 抖音+小红书（已审核通过的内容） |
| **13:00-16:00** | 生产明天的内容 | AI批量生产Day N+1的全部内容 |
| **16:00-17:00** | 内部质检 | 合规检查+差异化检查+品牌一致性 |
| **17:00** | 提交品牌方审核 | 批量提交明天要发的所有内容 |
| **19:00-21:00** | 第二波发布+互动 | 抖音+视频号（已审核通过的内容） |
| **22:00-23:00** | 数据复盘 | 回收今日数据+确定后天的内容方向 |

### 2.2 提交审核的内容格式建议

向品牌方提交审核时，建议用统一的表格格式（飞书/腾讯文档共享）：

| 内容ID | 平台 | 账号 | 类型 | 脚本文案 | 配图/视频链接 | 审核状态 | 品牌方备注 |
|--------|------|------|------|---------|------------|---------|----------|
| D3-DY-HH-01 | 抖音 | 灏灏 | 视频 | （完整文案） | （云盘链接） | 待审核 | |
| D3-XHS-JH-02 | 小红书 | 家豪 | 图文 | （完整文案） | （图片链接） | 待审核 | |

品牌方只需在"审核状态"列填：通过 / 打回 / 需修改（附备注）

## 三、内容库存管理

### 3.1 库存分级

```
一级库存（当日即发）：已通过品牌审核，今天发布
二级库存（次日待发）：今天生产完毕，已提交审核中
三级库存（应急备用）：之前生产但未使用的内容，随时可发
```

### 3.2 安全库存要求

| 阶段 | 每日发布量 | 安全库存量 | 说明 |
|------|---------|---------|------|
| 第一周 | 55-75条/天 | 额外备20条 | 品牌方可能打回较多（初期磨合） |
| 第二周 | 40-55条/天 | 额外备15条 | 审核通过率上升 |
| 第三周 | 70-95条/天 | 额外备25条 | 冲刺期不能断供 |

### 3.3 备用内容的生产策略

每周一固定生产一批"万能备用内容"——不依赖时效性，任何时候都能发：
- 品牌故事型（王老吉百年传承+世宝吉高端定位）
- 原料科普型（茯苓/化橘红/枸杞知识科普）
- 产品展示型（纯产品视觉展示+简单口播）

这些内容提前一次性审核通过，存入三级库存，随时可用。

### 3.4 内容库存追踪表

每天维护一张库存表：

| 日期 | 一级库存（已审核） | 二级库存（审核中） | 三级库存（备用） | 今日发布消耗 | 剩余 |
|------|----------------|----------------|---------------|-----------|------|
| Day 3 | 60条 | 55条 | 20条 | 50条 | 一级剩10 |
| Day 4 | 55+10=65条 | 60条 | 18条 | 55条 | 一级剩10 |

## 四、品牌方审核的沟通策略

### 4.1 前3天：建立审核标准

- Day 1 提交第一批内容时，主动附上一份"内容创作规范"给品牌方确认
- 规范中包含：允许使用的话术、禁止用语、品牌logo使用规范
- 请品牌方明确：哪些内容"一定会被打回"——提前规避
- 目标：让品牌方的审核标准可预测

### 4.2 提高审核通过率的技巧

- 所有文案严格使用品牌方认可的"安全话术库"
- 每条内容都标注引用了哪条安全话术（方便品牌方快速确认）
- 产品图/视频中不出现与品牌方审美不符的元素
- 如果品牌方反复打回某类内容，立刻停止生产该类内容

### 4.3 加速审核的方法

- 与品牌方约定固定审核时间（如每天17:00提交，次日10:00前返回）
- 按优先级排序提交：先提交次日必发的内容，备用内容最后
- 建立品牌方审核人的微信直接沟通渠道，紧急内容可加急
- 第二周后如果信任建立，争取"模板预审核"——某个模板通过后，该模板的变体可以免审或快审

## 五、Day 0 和 Day 1 的特殊安排

因为T+1的约束，项目的真正"第一天"应该是Day 0：

### Day 0（4月2日）：纯生产日

- 生产Day 1-2要发的全部内容（养号期内容+首批测试内容）
- 全部提交品牌方审核
- 同时生产一批"万能备用内容"提前审核

### Day 1（4月3日）：正式启动

- 上午接收审核结果
- 开始发布养号内容（不挂车的纯内容）
- 同时生产Day 2的内容并提交审核

### Day 2-3（4月4-5日）

- 发布Day 1审核通过的内容
- 生产Day 3-4的内容
- 开始小比例挂车测试

**这意味着你的21天实际生产周期是Day 0 - Day 22，但发布周期是Day 1 - Day 23。总共24天的工作量，不是21天。**

## 六、被品牌方打回的应急预案

### 6.1 如果打回率 < 20%（正常情况）

- 从三级备用库存中补充
- 被打回的内容修改后重新提交（争取当天下午加急审核）

### 6.2 如果打回率 20%-50%（需要警觉）

- 分析被打回的共性原因
- 可能是某个模板/话术整体不符合品牌方预期
- 立刻调整Claude的系统提示词，增加品牌方的具体审核偏好
- 动用全部三级库存补缺

### 6.3 如果打回率 > 50%（危机）

- 暂停生产，紧急与品牌方开会对齐审核标准
- 请品牌方提供"合格内容范例"
- 按范例重新调整全部内容方向
- 这可能导致1-2天的发布中断——用三级库存撑过去
