# AI生产系统操作手册｜端到端自动化内容工厂

> 本手册告诉生产系统"如何生产"
> 目标：实现从脚本生成→视觉制作→视频合成→发布→数据回收的全流程自动化
> AI是主力，人类只做质检、发布确认和策略决策

---

```

└──────────────┬───────────────────────────────────────────┘

│              Stage 2: 视觉引擎                             │
│  ┌─────────────────┐    ┌─────────────────┐              │
│  │ Nano Banana 2.0  │    │ Seedance 2.0     │             │

│  │ 封面图/场景图      │    │ 产品展示/动效     │             │
│  └─────────────────┘    └─────────────────┘              │

               ↓

### 2.2 每日脚本生产流程

**上午9:00 启动生产：**

1. 根据昨晚复盘结论，确定今日内容方向
2. 将方向填入系统提示词的"今日生产指令"
3. 分5轮生产（每人负责自己3个账号的脚本）：
   - 灏灏：10条（抖音科普5 + 视频号养生3 + 小红书成分2）
   - 家豪：10条
   - 小瓶子：10条
   - 和煦好物：10条
   - 泽鑫：10条
4. 每轮出10条脚本，共50条/天
5. 人工10分钟快速浏览，剔除明显不好的，保留35-40条进入下一阶段

### 3.1 Nano Banana 2.0 图片生成指令库

旁边有笔记本电脑和一杯倒出的淡金色液体，

```

```
提示词方向：世宝吉礼盒放在红色绸缎背景上，
旁边有干花和红色丝带装饰，
中式轻奢风格，温暖的暖色灯光，
拍摄角度略俯拍，突出礼盒质感和品牌logo

**场景3：原料展示（信任感场景）**

提示词方向：客厅茶几上放着世宝吉产品，

传递全家四季常备的感觉
```

### 3.2 Seedance 2.0 视频生成指令库

**动效1：产品360度展示**
```
输入：产品正面高清图
指令：将产品做缓慢360度旋转展示，
背景纯色或简约渐变，灯光从上方打下，
突出金色瓶身/罐身的质感和龙纹浮雕
时长：8-10秒
```

**动效2：倒饮效果**
```
输入：杯子+饮料的静态图
指令：模拟液体从瓶/罐倒入透明玻璃杯的动态，
液体为淡金色/琥珀色，有微微的光泽感，
镜头从侧面拍摄，突出液体的通透质感
时长：5-8秒
```

**动效3：礼盒开箱**
```
输入：礼盒正面图
指令：模拟礼盒盖缓慢打开，
内部产品逐渐露出，有柔光照射，
营造"惊喜揭晓"的感觉
时长：6-8秒
```

**动效4：原料到产品的转化**
```
输入：原料图（茯苓/枸杞等）+ 产品图
指令：原料图渐变过渡到成品产品图，
中间可以有"熬制""融合"的意象动效，
传递从天然原料到成品的匠心过程
时长：8-10秒
```

### 3.3 品牌视觉规范（所有AI图片/视频必须遵守）

- 主色调：品牌红(#C92029) + 鎏金(#FFF500) + 本草绿(#2E7D32) + 米白(#F5F0E6)
- 设计风格：新中式轻奢 + 国潮养生风
- 视觉：简约大气、留白高级、线条雅致
- 禁止：廉价感的配色、过于花哨的特效、与品牌调性不符的风格
- 字体：标题使用中式衬线字体，正文使用简约无衬线字体

## 四、Stage 3 详解：合成引擎（剪映）

### 4.1 视频模板设置（在剪映中预设好）

**模板参数：**
- 画幅：9:16竖屏（抖音/视频号）或 3:4（小红书视频）
- 时长：15-30秒
- 字幕：白色描边字幕，字号适中，位于画面下1/3
- BGM：轻音乐/国风轻音乐，音量为口播的30%
- logo水印：世宝吉logo，右上角或左下角，半透明
- 片尾：1-2秒品牌露出+引导点击购买

### 4.2 配音规范

| 账号人设 | 推荐音色 | 语速 | 情感 |
|---------|---------|------|------|
| 本草科普（灏灏） | 知性女声 | 中速 | 沉稳专业 |
| 办公室养生（家豪） | 年轻男声 | 稍快 | 活泼亲切 |
| 生活好物（小瓶子） | 温柔女声 | 中速 | 温暖种草 |
| 测评家（和煦好物） | 磁性男声 | 中速 | 理性可信 |
| 送礼情报（泽鑫） | 成熟男声 | 中速 | 大气稳重 |

### 4.3 批量合成流程

```
1. 导入Seedance生成的视频片段（多段）
2. 按脚本顺序排列片段
3. 套用预设模板（字幕+BGM+转场+logo）
4. 导入AI配音音频，对齐画面节奏
5. 检查时长（15-30秒），必要时剪裁
6. 批量导出
7. 按"日期-平台-账号-编号"命名（如 D7-DY-HH-03）
```

## 五、Stage 4 详解：质检门

### 5.1 质检清单（每条内容发布前30秒过一遍）

```
□ 合规检查：是否包含禁用词？（特别注意AI可能生成的"润肺""调理"等词）
□ 产品信息：产品名称/价格是否正确？
□ 品牌表述：是否说"王老吉旗下"（正确）而不是其他不准确的表述？
□ 差异化：与今天已发的内容是否有足够区别？
□ 画面质量：AI生成的画面是否有明显瑕疵/畸形？
□ 配音同步：配音与画面节奏是否匹配？
□ 商品链接：挂车链接是否正确？点击能否正常跳转？
□ 标签：是否带上了必要的话题标签？
```

### 5.2 质检不通过的处理

- 违禁词：直接打回重新生成（不能手动改一个词就发，要整条重做）
- 画面瑕疵：重新用Seedance生成或用备用素材替换
- 重复度高：更换hook和配音，重新合成
- 链接错误：修复后重新确认

### 6.3 数据驱动的自动决策规则

| 指标组合 | 判定 | 下一步动作 |
|---------|------|----------|
| 完播率>30% + 点击率>3% + 有出单 | 放大 | 投DOU+，批量生产变体 |
| 完播率>25% + 点击率>2% + 无出单 | 观察 | 检查详情页，优化CTA |

| 完播率<15% | 淘汰 | hook无效，换方向 |
| 播放>5000 + 零出单 | 检查 | 可能是链接/详情页问题 |

## 七、小红书图文专项生产流程

小红书图文不走视频流水线，走专属图文流程：

```
Claude生成文案（标题+正文+标签）
      ↓
Nano Banana生成封面图（第一张图决定点击率）
      ↓
Nano Banana生成2-4张内容配图（产品细节、使用场景、原料等）
      ↓
人工排版（加文字贴纸、品牌元素）
      ↓
质检（合规词 + 标签检查）
      ↓
发布
```

**小红书封面图公式：**
- 左侧或中心放产品大图
- 右侧或上方放1行标题文字（大号、醒目）
- 背景简洁、暖色调
- 整体风格精致高级，不廉价

## 九、AI内容与实拍内容的混合比例

纯AI内容有被平台限流的风险，建议按以下比例混合：

| 阶段 | 纯AI内容 | AI+实拍混合 | 纯实拍 |
|------|---------|-----------|--------|
| 第一周 | 60% | 30% | 10% |
| 第二周 | 50% | 40% | 10% |
| 第三周 | 40% | 45% | 15% |

**实拍素材获取方式：**
- 手机拍摄产品实物（开箱、倒饮、放在桌上的日常画面）
- 每人每周花30分钟拍一批实拍素材，存入共享素材库
- 这些实拍片段混入AI视频中，降低AI痕迹

## 十、生产系统的渐进式自动化路径

### 10.1 第一周：半自动化（学习阶段）

- Claude生成脚本 → 人工筛选 → AI生成视觉 → 人工合成 → 人工质检 → 人工发布
- 自动化程度：约40%
- 目标：跑通全流程，验证每个环节

### 10.2 第二周：高度自动化（优化阶段）

- Claude按固定prompt批量生成 → 快速筛选 → AI批量生成视觉 → 剪映模板批量套用 → 快速质检 → 定时发布
- 自动化程度：约70%
- 目标：缩短单条内容生产时间到15分钟以内

### 10.3 第三周：接近全自动化（产出阶段）
